nekoclaude Architecture

Local FastAPI web UI to manage Claude Code configs across machines

Strict layering

  • • Routers never open() a target file
  • • All mutations go through apply.py
  • • Services own the fs boundary
  • • No database — just files in catalog/, logs/, state/

Snapshot-first mutations

  • • Before any write, touched paths copied to logs/<t>/snapshots/<cid>/
  • • history.jsonl appended after the write
  • • Per-change revert restores from the snapshot
  • • Reverts themselves are revertible

Frontend without a build

  • • HTMX 1.9 + Alpine 3 + Tailwind via CDN
  • • Server returns HTML partials, not JSON
  • • Errors via HX-Retarget into #editor-error
  • • Active tab persisted in localStorage

nekoclaude · single-user, single-process · bind to 127.0.0.1 or trusted-tailnet only
